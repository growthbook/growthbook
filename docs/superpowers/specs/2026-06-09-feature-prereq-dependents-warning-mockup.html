2 · REST API

GET /api/v2/features/checkout-flow new dependents object on single-feature responses (v1 & v2) 200 OK 
"feature": {
  "id": "checkout-flow",
  "archived": false,
  "project": "prj_ecom",
  "valueType": "boolean",
  "defaultValue": "true",
  "owner": "jess",
  "ownerEmail": "jess@acme.com",
  "tags": ["checkout"],
  "environments": { /* … unchanged … */ },
  "prerequisites": [],
  "revision": { "version": 12, "comment": "" },
  "dependents": {                          // ← NEW
    "features": ["checkout-banner", "express-pay", "one-click-upsell"],
    "experiments": [
      { "id": "exp_8f2a", "name": "Checkout button color test" }
    ]
  }
}
POST /api/v2/features/checkout-flow non-blocking warnings[] on the update envelope (v1 & v2) 200 OK 
{
  "feature": { "id": "checkout-flow", "defaultValue": "false", /* … */
              "dependents": { "features": ["…"], "experiments": ["…"] } },
  "warnings": [                                // ← NEW, only when dependents exist
    {
      "type": "prerequisiteDependents",
      "message": "This feature is a prerequisite for 3 other features and 1 experiment. Your change may affect them."
    }
  ]
}
DELETE /api/v1/features/checkout-flow blocked when prerequisite for others — v1 & v2 400 Bad Request 
{
  "message": "Cannot delete 'checkout-flow' — it is a prerequisite for 3 features (checkout-banner, express-pay, one-click-upsell) and 1 experiment (Checkout button color test). Remove these prerequisite references first."
}
POST /api/v1/features/checkout-flow archiving (archived: true) is blocked the same way — v1 & v2 400 Bad Request 
// request body
{ "archived": true }

// response
{
  "message": "Cannot archive 'checkout-flow' — it is a prerequisite for 3 features (checkout-banner, express-pay, one-click-upsell) and 1 experiment (Checkout button color test). Remove these prerequisite references first."
}

Mockup only — copy, counts and IDs are illustrative. Pluralization (“1 feature” / “3 features”), show/hide of the dependents list, and the exact warnings wording are finalized in implementation.
