# Slack integration

The GrowthBook Slack integration sends GrowthBook event notifications to Slack
channels. Each connected Slack channel creates a Slack-formatted Event Webhook
in GrowthBook, so delivery status, retries, logs, and notification filters use
the same system as other Event Webhooks.

## Connect your Slack workspace

1. In GrowthBook, go to **Integrations → Slack**.
2. Click **Connect to Slack** and approve the GrowthBook app for your
   workspace.
3. After Slack redirects back to GrowthBook, add the channels GrowthBook
   should post to.

## Add channels

Once the workspace is connected, click **Add channel** on the Slack
integrations page and pick a channel from the searchable list. GrowthBook
joins public channels automatically.

For **private channels**, first run `/invite @GrowthBook` in the channel in
Slack — it then appears in the channel picker.

Each channel is configured independently (events, filters, digests), so
different channels can receive different updates.

## Configure notifications

Slack connections are backed by Event Webhooks with the `Slack` payload type.
From the Slack integrations page, use **Configure Slack alerts** to quickly
toggle common alert groups for each channel:

- **Lifecycle**: experiment started, experiment stopped and shipped, and
  experiment stopped and rolled back.
- **Health**: SRM warnings, multiple exposure warnings, guardrail failures,
  results updated with no data, and query failures.
- **Decisions**: ship, rollback, review, and metric significance alerts.

Open **Configure** for a channel to manage advanced Event Webhook filters:

- **Event filters**: choose which GrowthBook events should post to Slack.
- **Environment filters**: limit notifications to specific environments.
- **Project filters**: limit notifications to specific GrowthBook projects.
- **Tag filters**: limit notifications to events with matching tags.
- **Enabled**: pause or resume delivery for the channel.

The experiment alert events available for Slack include:

- `experiment.started`
- `experiment.stopped.shipped`
- `experiment.stopped.rolledback`
- `experiment.warning`
- `experiment.health.guardrailFailed`
- `experiment.health.noData`
- `experiment.health.queryFailed`
- `experiment.info.significance`
- `experiment.decision.ship`
- `experiment.decision.rollback`
- `experiment.decision.review`

## Coalesced notifications

When several events fire for the same object (experiment or feature) within a
short window, GrowthBook combines them into a single Slack message instead of
posting each one separately. For example, a failed bandit refresh that emits a
query-failed event plus a follow-up internal `experiment.updated` will arrive
as one digest message that lists both changes.

- The default window is 15 seconds and applies to newly connected Slack
  channels. Existing webhooks keep delivering one Slack message per event
  until the window is set on them.
- The window keys on the touched object: events about different experiments do
  not get merged with each other.
- The first event's arrival time + window determines when the digest is sent;
  the window does not extend as more events arrive, so a steady stream still
  produces a digest every window length.
- Up to five events are shown in full; the rest are summarized as "+N more
  changes not shown".
- Webhook test deliveries always fire immediately, never coalesced.

To change or disable the window for a channel, set `coalesceWindowMs` on the
underlying Event Webhook (0 disables coalescing). Raw and JSON Event Webhooks
keep their one-to-one event mapping regardless of this setting so API consumers
see every event.

## Testing your alerts

Open the Event Webhook for the Slack channel and click **Test**. You can also
trigger one of the events selected in your filters, such as creating a test
feature and then deleting it.

## Self-hosted setup

GrowthBook Cloud is already configured for Slack OAuth. If you are self-hosting,
create a Slack app and configure these settings before users can connect
channels:

1. Create a Slack app from your workspace's app dashboard.
2. In **OAuth & Permissions**, add these bot scopes:
   `chat:write`, `files:write`, `channels:read`, `groups:read`,
   `channels:join`, `commands`, `users:read`, `users:read.email`,
   `app_mentions:read`, `channels:history`, `groups:history`, `im:history`,
   `mpim:history`, `links:read`.
3. Add `{APP_ORIGIN}/integrations/slack` as an OAuth redirect URL, replacing
   `{APP_ORIGIN}` with your GrowthBook app URL.
4. Set `SLACK_CLIENT_ID` and `SLACK_CLIENT_SECRET` on the GrowthBook API server.

Installing is workspace-level — Slack shows no channel picker during OAuth.
Delivery uses the bot token (`chat.postMessage` and private file uploads);
channels are selected in GrowthBook after connecting. Connections created by
older GrowthBook versions with the `incoming-webhook` scope keep working
unchanged.
