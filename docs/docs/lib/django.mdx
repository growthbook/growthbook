## Quick Usage

The simplest way to add GrowthBook to a standard WSGI Django project is a middleware that creates a fresh `GrowthBook` instance for every request, attaches it to `request`, and cleans up after the response is sent.

### Configure the SDK key

Read `GROWTHBOOK_CLIENT_KEY` from the environment so it is never committed to source control:

```python title="settings.py"
import os

GROWTHBOOK_CLIENT_KEY = os.environ["GROWTHBOOK_CLIENT_KEY"]
```

### Create the middleware

:::warning Performance note
`load_features()` makes a blocking HTTP request to the GrowthBook CDN. To avoid adding a CDN round-trip to every response, cache the feature payload with Django's cache framework (see [caching features](#caching-features)) or use the [async client](/lib/python#async-client-recommended) with an ASGI server instead.
:::

```python title="myapp/middleware.py"
from django.conf import settings
from growthbook import GrowthBook


def on_experiment_viewed(experiment, result, **kwargs):
    # Send to your analytics backend
    print(f"Experiment: {experiment.key}, Variation: {result.key}")


def growthbook_middleware(get_response):
    def middleware(request):
        request.gb = GrowthBook(
            api_host="https://cdn.growthbook.io",
            client_key=settings.GROWTHBOOK_CLIENT_KEY,
            on_experiment_viewed=on_experiment_viewed,
            attributes={
                # Stable identifier for bucketing — use a UUID or database PK,
                # not a session key, so assignments persist across sessions.
                "id": str(request.user.pk) if request.user.is_authenticated else (request.session.session_key or "anonymous"),
                "loggedIn": request.user.is_authenticated,
                "employee": getattr(request.user, "is_staff", False),
                "country": request.META.get("HTTP_CF_IPCOUNTRY", ""),
                "url": request.build_absolute_uri(),
            },
        )
        request.gb.load_features()
        try:
            response = get_response(request)
        finally:
            request.gb.destroy()  # releases memory; always call this
        return response

    return middleware
```

### Caching features

Calling `load_features()` on every request hits the CDN each time. Cache the payload with Django's cache framework to fetch it once and reuse it across requests:

```python title="myapp/middleware.py"
from django.conf import settings
from django.core.cache import cache
from growthbook import GrowthBook

FEATURES_CACHE_KEY = "growthbook_features"
FEATURES_CACHE_TTL = 60  # seconds


def growthbook_middleware(get_response):
    def middleware(request):
        features = cache.get(FEATURES_CACHE_KEY)
        request.gb = GrowthBook(
            api_host="https://cdn.growthbook.io",
            client_key=settings.GROWTHBOOK_CLIENT_KEY,
            features=features,  # None on first request — load_features() fetches and stores it
            attributes={
                "id": str(request.user.pk) if request.user.is_authenticated else (request.session.session_key or "anonymous"),
                "loggedIn": request.user.is_authenticated,
            },
        )
        if features is None:
            request.gb.load_features()
            cache.set(FEATURES_CACHE_KEY, request.gb.features, FEATURES_CACHE_TTL)
        try:
            response = get_response(request)
        finally:
            request.gb.destroy()
        return response

    return middleware
```

### Register the middleware

Add the middleware to `MIDDLEWARE` in `settings.py`, after `AuthenticationMiddleware` so that `request.user` is already populated:

```python title="settings.py"
MIDDLEWARE = [
    "django.middleware.security.SecurityMiddleware",
    "django.contrib.sessions.middleware.SessionMiddleware",
    "django.middleware.common.CommonMiddleware",
    "django.middleware.csrf.CsrfViewMiddleware",
    "django.contrib.auth.middleware.AuthenticationMiddleware",  # must be before GrowthBook
    "django.contrib.messages.middleware.MessageMiddleware",
    "django.middleware.clickjacking.XFrameOptionsMiddleware",
    "myapp.middleware.growthbook_middleware",  # add after auth
]
```

:::warning Middleware ordering
If `growthbook_middleware` is placed before `AuthenticationMiddleware`, `request.user` will be an `AnonymousUser` for every request and your user-based targeting will not work.
:::

### Use it in views

`request.gb` is available in every view:

```python title="myapp/views.py"
from django.shortcuts import render


def home(request):
    show_new_banner = request.gb.is_on("new-banner")
    button_color = request.gb.get_feature_value("button-color", "blue")

    return render(request, "home.html", {
        "show_new_banner": show_new_banner,
        "button_color": button_color,
    })
```

And in templates:

```html title="templates/home.html"
{% if show_new_banner %}
  <div class="banner">Try our new feature!</div>
{% endif %}

<button style="background-color: {{ button_color }}">Click me</button>
```

## Integration Patterns

The three patterns below all use the same GrowthBook SDK. What differs is the instantiation point and instance lifetime. Pick the one that matches where the decision is made in your application:

| Pattern | Where the instance lives | Use when |
|---|---|---|
| Permissions layer | Per request, inside a permission check | A flag controls *access* |
| Async / Django Channels | Per WebSocket connection | Real-time features, chat, live updates |
| Background tasks | Per task invocation | Scheduled jobs, Celery tasks, management commands |

---

## Pattern 1 – Permissions layer

When a feature flag controls whether a user can *access* a resource, put the GrowthBook evaluation in the permission layer. The access decision stays explicit and separate from view logic.

### Plain Django – class-based views

`dispatch()` on a `View` subclass runs before any HTTP method, making it the right place to check the flag:

```python title="myapp/views.py"
from django.conf import settings
from django.http import HttpResponseForbidden
from django.views import View
from growthbook import GrowthBook


class BetaFeatureView(View):
    def dispatch(self, request, *args, **kwargs):
        gb = GrowthBook(
            api_host="https://cdn.growthbook.io",
            client_key=settings.GROWTHBOOK_CLIENT_KEY,
            attributes={
                "id": str(request.user.pk) if request.user.is_authenticated else "anonymous",
                "beta": request.user.groups.filter(name="beta").exists() if request.user.is_authenticated else False,
            },
        )
        gb.load_features()

        if not gb.is_on("beta-feature"):
            gb.destroy()
            return HttpResponseForbidden("This feature is not available yet.")

        request.gb = gb
        try:
            response = super().dispatch(request, *args, **kwargs)
        finally:
            gb.destroy()
        return response

    def get(self, request):
        return render(request, "beta.html", {})
```

### Plain Django – function-based views

A decorator keeps the access check reusable across function-based views:

```python title="myapp/decorators.py"
from functools import wraps
from django.conf import settings
from django.http import HttpResponseForbidden
from growthbook import GrowthBook


def require_feature(flag_key):
    def decorator(view_func):
        @wraps(view_func)
        def wrapper(request, *args, **kwargs):
            gb = GrowthBook(
                api_host="https://cdn.growthbook.io",
                client_key=settings.GROWTHBOOK_CLIENT_KEY,
                attributes={
                    "id": str(request.user.pk)
                    if request.user.is_authenticated
                    else (request.session.session_key or "anonymous")
                },
            )
            gb.load_features()
            enabled = gb.is_on(flag_key)
            gb.destroy()

            if not enabled:
                return HttpResponseForbidden("Feature not available.")
            return view_func(request, *args, **kwargs)

        return wrapper
    return decorator
```

```python title="myapp/views.py"
from myapp.decorators import require_feature


@require_feature("new-checkout")
def checkout(request):
    ...
```

### Django REST Framework

With DRF, a `BasePermission` subclass is the natural integration point. When `has_permission` returns `False`, DRF responds with a 403:

```python title="myapp/permissions.py"
from django.conf import settings
from rest_framework.permissions import BasePermission
from growthbook import GrowthBook


class FeatureFlagPermission(BasePermission):
    """
    Denies access when a GrowthBook feature flag is disabled.

    Usage:
        class MyView(APIView):
            permission_classes = [IsAuthenticated, FeatureFlagPermission]
            growthbook_flag = "my-feature"
    """

    def has_permission(self, request, view):
        flag_key = getattr(view, "growthbook_flag", None)
        if flag_key is None:
            return True  # no flag configured, allow through

        gb = GrowthBook(
            api_host="https://cdn.growthbook.io",
            client_key=settings.GROWTHBOOK_CLIENT_KEY,
            attributes={"id": str(request.user.pk), "loggedIn": request.user.is_authenticated},
        )
        gb.load_features()
        result = gb.is_on(flag_key)
        gb.destroy()
        return result
```

```python title="myapp/views.py"
from rest_framework.views import APIView
from rest_framework.permissions import IsAuthenticated
from rest_framework.response import Response
from myapp.permissions import FeatureFlagPermission


class BetaDashboardView(APIView):
    permission_classes = [IsAuthenticated, FeatureFlagPermission]
    growthbook_flag = "beta-dashboard"

    def get(self, request):
        return Response({"data": "beta dashboard content"})
```

## Pattern 2 – Async / Django Channels

For WebSocket consumers, the `GrowthBookClient` opens when the connection is established and closes when the connection is disconnected. The async client is designed for long-lived use: it refreshes features in the background without blocking the event loop.

Install the required packages:

```bash
pip install channels growthbook
```

```python title="myapp/consumers.py"
import json
from channels.generic.websocket import AsyncWebsocketConsumer
from growthbook import GrowthBookClient, Options, UserContext


class FeatureConsumer(AsyncWebsocketConsumer):
    async def connect(self):
        # Grab the authenticated user from the scope populated by AuthMiddlewareStack
        user = self.scope.get("user")

        from django.conf import settings
        self.gb_client = GrowthBookClient(
            Options(
                api_host="https://cdn.growthbook.io",
                client_key=settings.GROWTHBOOK_CLIENT_KEY,
            )
        )
        await self.gb_client.initialize()

        self.user_context = UserContext(
            attributes={
                "id": str(user.pk) if user and user.is_authenticated else "anonymous",
                "loggedIn": bool(user and user.is_authenticated),
            }
        )

        await self.accept()

        # Send the initial feature state on connect
        await self._send_features()

    async def disconnect(self, close_code):
        # Always close the client to release background tasks and connections
        if hasattr(self, "gb_client"):
            await self.gb_client.close()

    async def receive(self, text_data=None, bytes_data=None):
        data = json.loads(text_data or "{}")

        if data.get("type") == "refresh_features":
            await self._send_features()

    async def _send_features(self):
        show_new_ui = await self.gb_client.is_on("new-ui", self.user_context)
        chat_enabled = await self.gb_client.is_on("live-chat", self.user_context)

        await self.send(json.dumps({
            "type": "feature_update",
            "features": {
                "new_ui": show_new_ui,
                "chat_enabled": chat_enabled,
            },
        }))
```

The consumer connects to a URL pattern in your routing config:

```python title="myapp/routing.py"
from django.urls import re_path
from myapp.consumers import FeatureConsumer

websocket_urlpatterns = [
    re_path(r"ws/features/$", FeatureConsumer.as_asgi()),
]
```

:::note Instance lifetime
The `GrowthBookClient` instance lives for the duration of the WebSocket connection (potentially minutes or hours). The client refreshes features in the background automatically, so users in long-lived connections pick up flag changes without reconnecting.
:::

## Pattern 3 – Background tasks

Background tasks have no HTTP request and no authenticated user in scope. Attributes come explicitly from whatever context the task has available: a user ID from the job payload, a tenant identifier from the queue message, or a static environment identifier.

### Celery

Each task function gets its own `GrowthBook` instance. Sharing one instance across tasks is not safe:

```python title="myapp/tasks.py"
from celery import shared_task
from django.conf import settings
from growthbook import GrowthBook


def on_experiment_viewed(experiment, result, **kwargs):
    print(f"[celery] experiment={experiment.key} variation={result.key}")


@shared_task
def send_weekly_digest(user_id: int, user_email: str, is_premium: bool):
    gb = GrowthBook(
        api_host="https://cdn.growthbook.io",
        client_key=settings.GROWTHBOOK_CLIENT_KEY,
        on_experiment_viewed=on_experiment_viewed,
        attributes={
            "id": str(user_id),
            "email": user_email,
            "premium": is_premium,
        },
    )
    gb.load_features()

    try:
        use_new_template = gb.is_on("new-digest-template")
        subject = gb.get_feature_value("digest-subject-line", "Your weekly digest")

        if use_new_template:
            # render and send with new template using subject
            pass
        else:
            # render and send with old template using subject
            pass
    finally:
        gb.destroy()  # ensure cleanup even if sending fails
```

### Django management commands

Management commands run in their own process with no request context. The attributes come from whatever data source is available, often a queryset or a configuration value:

```python title="myapp/management/commands/run_experiment_report.py"
from django.conf import settings
from django.core.management.base import BaseCommand
from growthbook import GrowthBook


class Command(BaseCommand):
    help = "Runs the nightly experiment report for a given user segment"

    def add_arguments(self, parser):
        parser.add_argument("--segment", type=str, default="all")

    def handle(self, *args, **options):
        segment = options["segment"]

        gb = GrowthBook(
            api_host="https://cdn.growthbook.io",
            client_key=settings.GROWTHBOOK_CLIENT_KEY,
            attributes={
                # Use a stable string that represents this job's context,
                # not a runtime value like time.time(), so bucketing is consistent
                # across re-runs.
                "id": f"cron-report-{segment}",
                "segment": segment,
                "environment": "production",
            },
        )
        gb.load_features()

        try:
            use_new_report = gb.is_on("new-report-format")
            self.stdout.write(f"Using {'new' if use_new_report else 'old'} report format for segment={segment}")
            # ... report generation logic
        finally:
            gb.destroy()
```

The command runs the standard way:

```bash
python manage.py run_experiment_report --segment=premium
```
